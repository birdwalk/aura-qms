# Document Control Procedure (Synthetic — Demo Only)

## Purpose
To describe the controls for creating, reviewing, approving, distributing, and
retiring documented information within the QMS.

## Scope
Applies to all documented information required by ISO 9001:2015 and all documented
information determined by the organisation as necessary for the effectiveness of the QMS.

## Document Lifecycle

### Creation
- Documents must have a unique identifier (e.g., DOC-001)
- Author, title, scope, version number, and review date must be defined
- Documents start in DRAFT status

### Review and Approval
- All documents must be reviewed for adequacy prior to issue
- Documents require formal approval from designated authority before use
- Upon approval, status changes from DRAFT to APPROVED

### Distribution and Access
- Only approved versions shall be available at points of use
- Obsolete versions must be removed or clearly marked as superseded

### Review Cycle
- Approved documents must be reviewed at defined intervals (typically annually)
- Review due dates must be tracked and monitored
- Documents not reviewed by their due date enter OVERDUE_REVIEW status

### Document Status Definitions
| Status | Meaning |
|--------|---------|
| APPROVED | Current, reviewed, approved version in use |
| DRAFT | Under preparation or pending approval |
| OVERDUE_REVIEW | Approval valid but review date has passed |
| MISSING | Required document not yet created |
| SUPERSEDED | Replaced by newer version |
